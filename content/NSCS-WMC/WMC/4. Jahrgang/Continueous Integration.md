**Software Testing...**
- Wichtig für Unternehmen um Kosten einzusparen
- Absturz Ariane 5 (Rakete die nicht gut getestet wurde)
- Ziel: Korrektheit und Vollständigkeit der Software laufend prüfen

**Testarten**
- Manuelles Testen (Menschlicher Tester)
	- Testplan
	- Regelmäßige Abstände
- Automatisiertes Testen (Software prüft)
	- Werden von Entwicklern erstellt
	- Ausführung bei Buildprozess (CI/CD)
- Exploratives Testen (manuell)
	- Ohne definierten Testplan Software testen
	- Gut für frühe Projektphase oder als Ergänzung später
	- Vorteil: Einfach; Nachteil: Nur für erfahrene Tester

**Automatisierte Tests**
- Motivation: Manuell Testen fuckt ab, automatisch geht schnell und is nd tagesabhängig
- Unit-Tests: Testen eine einzige Komponente
- Integration-Tests: Testen Zusammenspiel mehrerer Komponenten
- System-Test: Testet komplett System
- Performance Test: logisch
- Lasttests: Hohe last testen
- Code Smells

- Ausführung durch Build-Management-Tools
