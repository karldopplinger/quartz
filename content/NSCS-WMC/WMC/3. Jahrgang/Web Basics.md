#wmc #matura #JS #HTML #CSS


**HTML (Hypertext Markup Language)** - definiert Grundstruktur, Ordnet Bilder, Texte etc. an
- Start-/Endtags </>
**CSS (Cascading Style Sheets)** - Legt aussehen fest (z.B Farbe, Größe von Schriften)
- style.css (Trennung von Inhalt & Design)
**JavaScript** - Interaktion mit Nutzer, Inhalte Dynamisch veränderbar
- Name nur Marketing weil Java bekannt

**Responsive Web Design**
- Reaktion auf Eigenschaften vom Endgerät (Auflösung, Touchscreen)
- Ziel: Benutzerfreundlichkeit auf allen Geräten
- Basis: HTML5, CSS3, JavaScript

**Sessions**
- Session ("Sitzung") - bestehende Client/Server-Verbindung
- Beginn: Login; Ende: Logout
**HTTP-Sessions**
- HTTP ist stateless (zustandslos) - Daten können nicht einfach gespeichert werden
- Lösung: Der Server speichert einen Cookie (session id) beim client und kann ihn so zu seinem Warenkorb zuordnen

**Cookies**
- Ermöglicht Speichern von Key/Value Pairs
- Werden von einer Domain gesetzt (Bei anfrage an die domain werden cookies automatisch mitgesandt)

**HTML5 Web Storage**
- Verbesserte Cookies (Mehr Speicherplatz)
- Daten müssen manuell abgefragt werden via JS
